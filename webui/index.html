Run metadata

Rendering

DPI for the initial pypdfium2 render. 200 dpi ≈ 1650×2200 for a US-letter page.

Diffusion input sizing

prepared.png = round_to_multiple_of_8(rendered.png × scale percent). Aspect ratio matches the rendered page exactly. If the resulting longer side exceeds long side max, both dimensions are scaled down proportionally so the longer side equals the cap — that way you can't accidentally launch a 2-hour-per-image job on a high-DPI render. Default guidance is only used when you click "+ add variant" below; each variant carries its own editable guidance.

Describe — questions for the VLM

Each page is described by asking InstructBLIP every question below, one at a time. The answers are stitched into a single paragraph used as the per-page LLM description. Variants in the table below decide individually whether to use that description as their negative prompt, or to override it with a manual one. Keep key short and alphanumeric (used in cache filenames like description_q01_key.txt).

Sequel variants

Each card is one img2img generation per page. The init image is always prepared.png. Leave seed blank to use the per-page seed (base_seed + page_idx*1009). Strengths are 0–1; higher = more deviation from the init. Per-variant guidance lets you sweep classifier-free guidance the same way you sweep strength and steps. Each variant also chooses its negative prompt: LLM (use the stitched description) or manual (type your own; empty string is allowed and means "no negative steering").
